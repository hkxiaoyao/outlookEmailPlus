## 一、背景与目标

你需要对一个 Flask + SQLite + 原生 JS/CSS 的 Outlook 邮箱管理系统进行前端 UI 重构。

**重构目标**：
1. 将当前「系统设置」页面（单页长滚动表单）改为 **4-Tab 导航结构**（基础 / 临时邮箱 / API 安全 / 自动化）
2. 在「临时邮箱」Tab 内将 GPTMail 和 Cloudflare Worker 两套配置**拆分为独立面板**，互不干扰
3. 将 Provider 选择器从 `<select>` 改为**单选按钮卡片**
4. CF Worker 同步域名字段改为**永久只读**（由同步 API 写入）
5. 非基础 Tab 切换时触发**自动保存**（PUT /api/settings），基础 Tab 密码字段排除

## 四、执行顺序

**请严格按 Phase 顺序执行，每个 Phase 完成后验证检查点再继续。**

### Phase 1：后端（约 30 分钟）

1. **`outlook_web/db.py`**：在 `temp_mail_prefix_rules` 的 INSERT 之后，新增 3 个 CF Worker key 的默认值：
   - `cf_worker_domains` → `'[]'`
   - `cf_worker_default_domain` → `''`
   - `cf_worker_prefix_rules` → 含默认规则的 JSON 字符串
   - 使用 `INSERT OR IGNORE`（不覆盖已有值）

2. **`outlook_web/repositories/settings.py`**：新增 3 个 getter 函数：
   - `get_cf_worker_domains()` → 返回 `list`，损坏 JSON 时返回 `[]`
   - `get_cf_worker_default_domain()` → 返回 `str`
   - `get_cf_worker_prefix_rules()` → 返回 `dict`，损坏 JSON 时返回默认 dict

3. **`outlook_web/controllers/settings.py` — GET 端点**：在 `safe_settings["temp_mail_prefix_rules"]` 之后暴露 3 个新 key

4. **`outlook_web/controllers/settings.py` — PUT 端点**：在 `temp_mail_prefix_rules` 处理之后，处理 3 个新 key 的写入（复用现有的 `_parse_temp_mail_domains_input` 和 `_parse_temp_mail_prefix_rules_input` 校验函数）

5. **`outlook_web/controllers/settings.py` — 同步接口**：将 `api_sync_cf_worker_domains` 函数中的写入目标从 `temp_mail_domains` / `temp_mail_default_domain` 改为 `cf_worker_domains` / `cf_worker_default_domain`

> 所有具体代码片段见 TODO 文档 Phase 1 各任务。

### Phase 2：前端 HTML 结构（约 60 分钟）

修改 `templates/index.html` 中的 `#page-settings` 区域：

1. **整体包裹**：用 `.settings-card-shell` 容器包裹 Tab 导航 + 所有 Tab 内容
2. **Tab 导航栏**：4 个按钮（基础 / 临时邮箱 / API 安全 / 自动化），用 `data-tab` 属性和 `onclick="switchSettingsTab('xxx')"` 控制
3. **基础 Tab** (`#settings-tab-basic`)：仅含密码字段 + 保存按钮
4. **临时邮箱 Tab** (`#settings-tab-temp-mail`)：
   - Provider 单选按钮卡片（两个 `.provider-radio`），替换旧 `<select id="settingsTempMailProvider">`
   - GPTMail 独立面板 `#gptmailConfigPanel`（默认隐藏）
   - CF Worker 独立面板 `#cfWorkerConfigPanel`（默认显示）
   - CF Worker 面板内：`settingsCfWorkerDomains`（只读 textarea）、`settingsCfWorkerDefaultDomain`（只读 input）为新增字段，**必须加 `readonly` 属性**
   - 同步时间展示元素 ID 为 `cfWorkerSyncTime`（旧 `cfWorkerSyncDomainsHint` 替换）
5. **API 安全 Tab** (`#settings-tab-api-security`)：将现有对外 API Key、公网安全配置、External Pool 配置**平移**至此 Tab，字段 ID 不变
6. **自动化 Tab** (`#settings-tab-automation`)：将现有 Token 刷新、自动轮询、Email 通知、Telegram 通知**平移**至此 Tab，字段 ID 不变

> ⚠️ **API Key 迁移注意**：将字段 `settingsApiKey` 改为 `settingsTempMailApiKey`，JS 中所有引用（含 `dataset.maskedValue`/`dataset.isSet`）需一并更新。

### Phase 3：前端 CSS 样式（约 20 分钟）

在 `static/css/main.css` 末尾追加以下样式块（代码已在 TODO 文档 Phase 3 中提供，可直接复制）：

1. **`.settings-card-shell`**：整体卡片外壳（border + border-radius + box-shadow）
2. **`.settings-tab-nav`**：Tab 导航栏，`background: var(--bg-secondary)`，`overflow-y: hidden`
3. **`.settings-tab`**：Tab 按钮基础样式，`:hover` 轻微背景，`.active` 底线为砖红色 `var(--clr-primary)` + 背景改为 `var(--bg-card)`
4. **`.settings-tab-pane`**：默认 `display: none`，`.active` 时 `display: block`
5. **`.settings-tab-pane .card`**：去掉独立 border/shadow，改用底部分割线
6. **`.tab-save-dot`**：保存中圆点，黄色 `var(--clr-warn)` 脉冲动画；`.save-error` 时红色 `var(--clr-danger)`
7. **`.provider-radio-group`** / **`.provider-radio`** 等：单选卡片样式，选中边框 `var(--clr-primary)`

> ⚠️ 所有 CSS 变量名使用 `var(--clr-primary)` 而非 `var(--primary)`！

### Phase 4：前端 JS 逻辑（约 90 分钟）

修改 `static/js/main.js`：

1. **`showToast` 扩展**：新增第 4 参数 `persistent = false`，为 `true` 时不自动消失，追加 × 关闭按钮

2. **新增 `switchSettingsTab(tabName)` 函数**：
   - 点击同一 Tab 无操作
   - **立即**切换 Tab 按钮 `.active` class 和内容区显隐（不等待保存）
   - 基础 Tab 切走时清空密码框（若有内容）并提示
   - 非基础 Tab 切走时调用 `autoSaveSettings(prevTab)`（不 await，后台异步）

3. **新增 `autoSaveSettings(tabName)` 函数**：
   - 根据当前 Tab 收集对应字段（临时邮箱 Tab 只收集当前 Provider 面板的可编辑字段）
   - 在上一个 Tab 按钮上显示**黄色圆点**（添加 `.saving` class）
   - 调用 `PUT /api/settings`（注意是 PUT，不是 POST）
   - 成功：移除圆点 + 显示成功 toast
   - 失败：圆点变红（`.save-error` class）+ 持久 toast（`persistent=true`）

4. **新增 `onTempMailProviderChange(provider)` 函数**：
   - `'legacy_bridge'` → 显示 `#gptmailConfigPanel`，隐藏 `#cfWorkerConfigPanel`
   - `'cloudflare_temp_mail'` → 反之

5. **修改 `loadSettings()` 函数**：
   - 加载 Provider radio 选中状态并调用 `onTempMailProviderChange`
   - 加载 `cf_worker_domains`、`cf_worker_default_domain` 到只读字段（**始终添加 `.readonly-field` class 和 `readonly` 属性**）
   - 加载 `cf_worker_prefix_rules` 到可编辑字段

6. **修改 `syncCfWorkerDomains()` 函数（约 2061 行）**：
   - 同步成功后写入 `settingsCfWorkerDomains`（不是 `settingsTempMailDomains`）
   - 同步成功后写入 `settingsCfWorkerDefaultDomain`（不是 `settingsTempMailDefaultDomain`）
   - 更新 `cfWorkerSyncTime` 元素（不是 `cfWorkerSyncDomainsHint`）
   - 写入后域名字段**保持只读**

## 五、关键约束与禁止事项
